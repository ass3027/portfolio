4-1 클로즈드 소스 바이너리 프로토콜 역공학 구현 ▾
문제
RPCN 서버는 공식 SDK가 없다. RPCS3 C++ 소스코드에서 15-byte 바이너리 헤더 + Protobuf 페이로드 구조를 분석하고 TLS/TCP 클라이언트를 직접 구현해야 했다.
15-byte Header <BHIQ> ├─ u8 pkt_type (Request=0 / Reply=1 / Notif=2) ├─ u16 command ├─ u32 total_size └─ u64 packet_id Payload: Simple → raw struct.pack LE integers Complex → u32 LE length prefix + Protobuf bytes
src/rpcn_client/client.py
def connect(self) -> int:
    ctx = ssl.create_default_context()
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE  # RPCN 자체 서명 인증서

    raw = socket.create_connection((self.host, self.port), timeout=30)
    self._sock = ctx.wrap_socket(raw, server_hostname=self.host)
    hdr = self._recv_exact(HEADER_SIZE)
    pkt_type, _cmd, pkt_size, _pkt_id = struct.unpack(_HDR_FMT, hdr)

def _recv_reply(self) -> tuple[int, bytes]:
    while True:
        hdr = self._recv_exact(HEADER_SIZE)
        pkt_type, cmd, pkt_size, _pkt_id = struct.unpack(_HDR_FMT, hdr)
        payload = self._recv_exact(pkt_size - HEADER_SIZE)
        if pkt_type == PKT_NOTIF:   # 서버 푸시 알림 스킵
            continue
        return payload[0], payload[1:]

def transactional(fn):
    @wraps(fn)
    async def wrapper(*args, **kwargs):
        async with async_session_factory() as session:
            async with session.begin():   # 성공 시 commit, 예외 시 rollback
                return await fn(*args, session=session, **kwargs)
    return wrapper

🔥 FireWatcher — 산불감지 시스템

개요

프로젝트 배경

해결
DeepStream 로그 에러 자동 수집 시스템 구축. FastAPI 기반 영상 테스트 시스템을 구현해 비개발 직군도 영상 상태와 에러를 직접 확인 가능하도록 했다. FFmpeg 옵션 최적화 및 영상 송출 프로세스 자동 재시작·로깅 추가.

2 DeepStream 모듈 리팩터링 — 클래스 분리 + 전역 변수 제거 ▾
문제
DeepStream 영상 분석 모듈이 단일 파일 구조와 전역 변수에 의존하여 유지보수와 신규 인력 온보딩이 어려운 상태였다.
해결
